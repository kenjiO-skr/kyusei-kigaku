# 暦データ源

## 方針（決定）
天文計算ライブラリを依存に加えず、**必要な暦点を JS テーブルに埋め込む**（静的Pages・依存最小・クライアント完結のため）。

## 必要な暦点と対象年範囲
| 暦点 | 用途 | 対象範囲 | 精度 |
|---|---|---|---|
| 立春（日付） | 本命星の年境界 | 1950–2035 | 日単位（時刻不要・日境界で判定） |
| 各月の節入り（日付） | 月命星・月盤の月境界 | 1950–2035（月命星）/ 2025–2035（月盤） | 日単位 |
| 冬至・夏至（日付） | 日盤の陰遁陽遁の基準 | 2025–2035 | 日単位 |
| 60干支日の甲子起点 | 日盤の中宮・干支日 | アンカー1点＋通日計算 | 日単位 |

注：本命星・月命星は「日付がどの節月に入るか」だけ分かればよく、時刻精度は不要。日盤は二至直近の甲子日を特定できればよい。

## 出典・検証（Phase 2 で確定）
- 一次的な暦データは国立天文台暦要項等の公表値に整合させる方針。
- Phase 2 の暦検証サブエージェント2体が、立春・節入り・二至・甲子・閏の境界ケースを多年分で相互反証し、確定値を `src/calc/calendar-data.js` に格納、本ファイルに出典と検証ログを追記する。
- 甲子の通日計算は既知のアンカー日（ユリウス通日 mod 60）から導出し、複数年の既知干支日と突合する。

## 干支日アンカー（Phase 2 で値を確定）
- 基準日とその干支番号（甲子=0）を1点固定し、対象日との通日差 mod 60 で算出する。アンカー値と検算結果はここに記録する。
