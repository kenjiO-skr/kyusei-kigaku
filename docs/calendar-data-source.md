# 暦データ源

## 方針（決定）
天文計算ライブラリを依存に加えず、**必要な暦点を JS テーブルに埋め込む**（静的Pages・依存最小・クライアント完結のため）。

## 必要な暦点と対象年範囲
| 暦点 | 用途 | 対象範囲 | 精度 |
|---|---|---|---|
| 立春（日付） | 本命星の年境界 | 1950–2035 | 日単位（時刻不要・日境界で判定） |
| 各月の節入り（日付） | 月命星・月盤の月境界 | 1950–2035（月命星）/ 2025–2035（月盤） | 日単位 |
| 冬至・夏至（日付） | 日盤の陰遁陽遁の基準 | 2025–2035 | 日単位 |
| 60干支日の甲子起点 | 日盤の中宮・干支日 | アンカー1点＋通日計算 | 日単位 |

注：本命星・月命星は「日付がどの節月に入るか」だけ分かればよく、時刻精度は不要。日盤は二至直近の甲子日を特定できればよい。

## 出典・検証（Phase 2 で確定 — 2026-05-29（令和8年5月29日））
確定値は `src/calc/calendar-data.js`（`SOLAR_TERMS` / `SOLSTICES` / `SEXAGENARY_ANCHOR`）に格納済み。

### 検証方法：独立2手法による相互反証
手法をあえて変えた2系統を突き合わせ、同じ誤りの共有を防いだ。
- **A体（計算）**：Meeus『Astronomical Algorithms』相当の太陽視黄経アルゴリズムで二十四節気（十二節）・二至の到達時刻を算出。TT→UT は ΔT（Espenak & Meeus 2006 多項式）、JST = UT + 9h で暦日を確定。系統残差は約2〜3分まで圧縮。
- **B体（転記）**：国立天文台（NAOJ）暦計算室の公表値を転記。中央標準時・日単位。

### 出典（B体・一次情報）
- NAOJ 二十四節気・雑節 長期版（1582年以降）：`https://eco.mtk.nao.ac.jp/cgi-bin/koyomi/cande/phenomena_sy.cgi`（`lst=9` 中央標準時 / `phenom=50` / EUC-JP）
- NAOJ 二十四節気・雑節 通常版（2009〜2027）：`https://eco.mtk.nao.ac.jp/cgi-bin/koyomi/cande/phenomena_s_en.cgi`
- NAOJ 暦要項PDF（1955〜2027）：`https://eco.mtk.nao.ac.jp/koyomi/yoko/pdf/yoko{YEAR}.pdf`
- NAOJ 暦要項HTML（2026）：`https://eco.mtk.nao.ac.jp/koyomi/yoko/2026/rekiyou262.html`

### 検証ログ（結果：全点一致・矛盾ゼロ）
- **立春 1950–2035（全86年）**：A体＝B体 完全一致。
  - 2/3 になる年：**2021・2025・2029・2033**（21世紀の前倒し。本範囲では2021が初出）
  - 2/5 になる年：**1951・1952・1956・1960・1964・1968・1972・1976・1980・1984**（うるう年系の遅れ。1984が最後）
  - 上記以外は全て 2/4。回帰テスト基準 **1971=2/4** も一致。
- **十二節 代表年（1950・1971・1984・2000・2025・2026）**：全節 A体＝B体 一致（B体は長期版CGI／PDFスキャンの二経路照合）。
- **二至 2025–2035**：A体＝B体 一致。夏至は全年 6/21、冬至は 12/21（うるう年前後）と 12/22 が交互。
- **境界リスク（JST 0時±3h、計263件）の極限ケース決着**：
  - **2025冬至 = 12/22**（A体計算では 12/22 00:00:21 の極限だが NAOJ も 12/22 → 確定）
  - 2021立春 = 2/3（23:58）・2025立春 = 2/3（23:09）・1984立春 = 2/5（00:21）：いずれも両班一致。
- A体の自己訂正：干支アンカーの初期値（俗説「1984-02-02=甲子」）は2日ズレの誤り。下記の通り確定し直した。

## 干支日アンカー（確定）
- **甲子（きのえね, index=0） ⟺ (JDN mod 60) === 11**。JDN は Fliegel–Van Flandern 整数式（正午基準）。
- 基準日：**2024-01-01（JST）= 甲子, JDN 2,460,311**（`SEXAGENARY_ANCHOR`）。
- 算出式：`index = (((jdn - 2460311) % 60) + 60) % 60`（0=甲子 … 59=癸亥）。
- 独立検算（別基準でも同位相を確認）：
  - A体：2000-01-01（JDN 2,451,545）= 戊午（54）。`(2451545 + 49) mod 60 = 54` ✓（49 ≡ −11 mod 60）。
  - B体：2024-03-01・2024-12-26・2026-02-19・2026-12-16 の既知甲子日が全て index=0 で一致（独立2ソース）。

## 日盤の陰遁陽遁＋閏（Phase 3b 確定 — 2026-05-30（令和8年5月30日））
実装は `src/calc/calendar.js` の `dayBoard`／`dayBoardCenter`。テストオラクルは `docs/verification-data.md` §4。

### 算出規則（符頭法・240日閏）
- 冬至＝陽遁(+1)開始（一白）／夏至＝陰遁(−1)開始（九紫）。切替日は二至に最も近い甲子。
- **九星の閏**：陽遁＋陰遁=360日に対し1年≈365.24日のズレを、11〜12年に一度60日延長して調整する。
  前の切替から**240日間隔**になる回だけ、甲子を30日前倒しして**甲午**にし（間隔210日）、
  冬至=七赤／夏至=三碧で開始する。結果、切替前後で同じ星が1日重複する。
- 1950–2035 の範囲で閏は **2031冬のみ**（2020夏・2043夏は範囲外）。

### 出典
- 国立天文台ベースの二至（本ファイル上記）＋干支日（SEXAGENARY_ANCHOR）から算出。
- 規則：Wikipedia「九星」、nobml「日家九星の計算方法」（`K=(MJD+50) mod 60`、符頭・閏）。
- オラクル：**yakumoin.info の公表日盤**（`https://yakumoin.info/daily/direction/date/YYYYMMDD`, 2000–2050）。

### 検証ログ（全点一致）
- 通常期：2026-02-04=一白／2026-06-21=七赤／2028-08-15=四緑／2030-07-01=三碧／2035-03-10=四緑。
- 2031冬の閏：12/19・12/20=七赤（**重複**）→12/22=九紫…2032-01-20=二黒まで連続一致。nobml の 2008/12/31=九紫（甲午・七赤の閏例）とも整合。
- 当初の per-solstice K 規則は閏（240日）を取り逃したため、**切替日チェーン＋240日判定**に修正して確定。

## その他の残課題・注意
- 単一経路のみの照合だった箇所（立春1950–1954、二至2028–2035の一部）も、A体の独立計算が一致しているため確定とする。
- 月命星・傾斜宮は仕様メモ§2/§3準拠で実装済だが独立数値オラクル未整備＝要確認。
- **SOLSTICES を将来拡張する際の注意（2026-06-12（令和8年6月12日）レビューで判明）**：
  - `rawSolsticeSwitch` の「最も近い甲子」タイブレーク（K=30＝二至当日が甲午で前後の甲子が等距離30日）は、
    現収録範囲 2025–2035 では出現せず（K は 1〜57）方向未検証。K=30 が出現する年を含めるときは
    yakumoin 等のオラクルでタイブレーク方向（前進/後退）を検証すること。
  - 閏検出（240日間隔判定）はチェーン先頭の切替には前間隔が存在せず適用できない。
    先頭年は実際の利用開始より1年以上前に取ること（例：2043夏の閏を含む拡張では2042年以前から収録）。
  - `dayBoard` は最後の切替日＋179日を上限とする範囲ガードを持つ。拡張時はテスト
    （tests/dayboard.test.js の上限境界）もあわせて更新すること。
