# ADR-002 運勢の多層化（本命／月命／傾斜／五行・同会）

- 状態：**確定**（2026年6月6日／令和8年6月6日）
- 決定者：本人（プロジェクトオーナー）
- 関連：機能②運勢、ADR-001（採用流派A＝園田系・標準）、`src/calc/fortune.js`

## 背景
従来の運勢（機能②）は **本命星の回座方位のみ**の単層で読んでいた（`fortuneOf`）。
週次・月次レビューでの読みに深みを持たせるため、本命に加えて
**月命星・傾斜宮・五行（同会）**を加味する多層化を行う。

九星気学では、月命星・傾斜宮を「運勢」にどこまで織り込むかに流派差があり
（園田系は本来これらを主に性格・相性に用い、年月日の運勢吉凶は本命中心に読む）、
また「同会」の取り方にも差がある。本 ADR でこの解釈選択を固定する。

## 決定

### 多層の構成（中庸B）
- **本命星＝基本運（主）**：従来どおり回座方位の象意（`fortuneOf` を不変のまま流用）。
- **月命星＝補助線**：同じ盤での月命星の回座方位の象意（内面・家庭・若年の補助）。
- **傾斜宮＝色付け**：出生固定の性格レンズ（盤で動かない。運気の強弱には足さない）。
- **五行（同会）＝運気の質**：本命星が回座した宮の**定位星**との五行関係。

いずれも UI 上で**「補助的な読み・流派により扱いが分かれる」旨を明示**する。

### 同会の定義
**同会星 ＝ 本命星が回座した宮の後天定位星**（`HOME_STAR_BY_DIR[本命の回座方位]`）。
本命の五行とこの定位星の五行の関係（`elementRelation`）で運気の質を読む。
（例：2026年盤で二黒は北西＝乾に回座 → 乾の定位星＝六白〔金〕→ 土生金＝退気。）
本命星が中宮（八方塞がり）のときは同会を取らず「守り」とする。

### 重み付け（論点①）→ **定性併記**
本命と月命を**比率合成しない／年齢補正も入れない**。本命を主・月命を補助線として
それぞれ別に提示する。非可証の理論に偽りの精度を持ち込まないため。

### 矛盾時（論点②）→ **併記・非合成**
本命（同会五行）と月命の運気の向きが「上向き×下向き」で割れたときのみ
`divergence.isSplit` を立て、割れている旨を注記表示する。**単一の総合判定（吉/凶の一語）は出さない**。

### 流派の範囲（論点③）→ **中庸B**
- A案（本命のみ・現状維持）：要望（深み）を満たさず不採用。
- **B案（採用）**：月命＝補助線、傾斜＝色付け、五行＝質。月命に独自の吉凶判定は作らない。
- C案（月命も独立吉凶・傾斜も加点減点）：園田系が認めない月命吉凶を創作するため不採用。

## 運気の向き（polarity）の対応
五行関係 → 向き：比和・生気＝上向き／退気＝横ばい／殺気・死気＝下向き／本命中宮＝守り。
割れ＝本命と月命が「上向き」と「下向き」で逆を向いたときのみ。

## 帰結（不変条件）
- **既存 `fortuneOf` のシグネチャ・戻り値は不変**。多層は新関数
  `fortuneLayers(honmei, getsumei, keisha, board)` として追加し、
  `engine.boardModel(date, honmei, period, birth)` の `birth` 指定時のみ `fortune.layers` に付与する。
- **ADR-001（吉方位の流派A）・§5/§2 オラクル・`direction.js`・`compatibility.js` は無改変**。
  本 ADR は運勢（機能②）の読みにのみ適用し、吉凶方位判定には影響しない。
- 本決定の変更は、この ADR の改訂と `tests/fortune.test.js`（多層運勢）の更新を前提とする。
