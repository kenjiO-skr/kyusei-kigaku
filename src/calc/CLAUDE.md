# CLAUDE.md — 暦計算・盤配置エンジン（src/calc）

このディレクトリは九星気学の**算出ロジックの心臓部**。UIから切り離し、純関数＋テーブルで構成する。UI都合の状態やDOM参照を持ち込まない。

## モジュール構成（予定）
- `constants.js` … 九星・五行・後天定位・八方位・十二支方位の定数テーブル
- `calendar-data.js` … 立春・各月節入り・二至・甲子起点の埋め込みテーブル（出典は `docs/calendar-data-source.md`）
- `calendar.js` … 日付→節月・立春境界判定・二至/甲子算出・日盤の陰遁陽遁＋閏
- `honmei.js` … 本命星・月命星・傾斜宮
- `board.js` … 各盤の中宮算出＋飛泊（盤配置）
- `fortune.js` … 本命星の回座位置から運勢
- `direction.js` … 凶殺判定＋吉方位判定（流派A）
- `compatibility.js` … 五行相性

## 不変条件（このディレクトリ固有）
1. **飛泊式は1箇所に集約**：`配置星 = ((d − 1 + (k − 5)) を9で正の剰余) + 1`（1〜9）。年/月/日盤で使い回す。重複定義禁止。
2. **マジックナンバー直書き禁止**：九星番号・方位・定位は `constants.js` の名前付き定数を参照。
3. **立春境界**：本命星算出で 1/1〜立春前日は前年。境界日の比較は「日付＋立春日時」で行い、`>=` / `<` の別を明示。
4. **正の剰余**：JS の `%` は負を返すため `((n % 9) + 9) % 9` で正規化してから +1。
5. **テストオラクルは `docs/verification-data.md`（§5）と `tests/`**。実装を変えて §5 が赤になったら実装が誤り。先にロジックを疑う。
6. **流派A固定**（`docs/adr-001-auspicious-school.md`）。吉方位の判定基準をここで変えない。

## 凶殺の定義（流派A・二黒土星）
- 五黄殺＝五黄回座方位／暗剣殺＝五黄の対冲／歳破・月破・日破＝その盤の十二支の対冲
- 本命殺＝本命星(二黒)の回座方位／本命的殺＝その対冲／八方塞がり＝本命星が中宮の盤
- 定位対冲・小児殺は**不採用**（流派A）

## 吉方位（流派A・二黒＝土）
- 凶殺非該当のうち、回座星が **火(九紫)=生気** または **土(二黒/八白)=比和** → **吉**
- 金(六白/七赤)=退気 → **中立**／木(三碧/四緑)=殺気・水(一白)=死気 → **小凶**
